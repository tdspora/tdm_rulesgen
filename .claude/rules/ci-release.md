# CI and Release Rules

- Preserve Python 3.11 and 3.12 compatibility unless explicitly changed (`requires-python = ">=3.11"`, ruff `target-version = "py311"`, mypy `python_version = "3.11"`).
- CI: GitHub Actions only (`.github/workflows/ci.yml`). Two jobs:
  - `test`: `uv sync --extra api --extra dev --locked`, then `ruff check`, `ruff format --check`, `mypy src`, `pytest`, `uv build`, `pip-audit`.
  - `release`: runs only on `main` / `master`, uses `python-semantic-release@v10.2.0`, requires baseline `v*` tag, uses `secrets.DEPLOY_KEY` for SSH push. Treat as high-risk.
- Versioning is owned by **`python-semantic-release`**:
  - Source of truth: `pyproject.toml:project.version` (declared in `[tool.semantic_release] version_toml`).
  - Bumps are computed from Conventional Commit history (`commit_parser = "conventional"`).
  - **Never manually edit `project.version`.** Use Conventional Commits — `feat:`, `fix:`, `chore:`, `refactor:`, `docs:`, `test:`, breaking change marker `!:` or `BREAKING CHANGE:` footer.
  - `CHANGELOG.md` is generated by semantic-release (`mode = "update"`); do not hand-edit beyond merge conflict resolution.
- Dependency changes:
  - Update via `uv add` / `uv remove` / `uv lock`; commit `pyproject.toml` and `uv.lock` together.
  - The CI step `uv sync --locked` will fail if the lockfile is out of sync.
  - Keep dependency additions minimal and justified.
- Validate packaging changes locally: `uv build` (produces `dist/*.whl` and `dist/*.tar.gz`). Inspect with `python -m zipfile -l dist/*.whl` to confirm no secrets or `.rulesgen-data/` slipped in.
- Docker / compose changes affect runtime topology and OpenSandbox integration (`compose.opensandbox.yaml`); treat as high-risk and escalate.
- Do not modify the release job, `DEPLOY_KEY` usage, or `semantic_release` configuration without explicit approval.
- Do not run `semantic-release` or `uv publish` from a developer machine — release is GitHub-Actions-only.
- `pip-audit` runs `continue-on-error: true` in CI but findings should still be triaged before release.
